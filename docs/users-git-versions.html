Recommended Git Versions

Shows how many users signed in with recommended, unrecommended, and vulnerable versions.

Vulnerable indicates that a Git version is affected by a publicly disclosed vulnerability. In this chart, affected Git versions are counted as vulnerable after the date the security issue was made public. On and before that date, the Git version is not considered as vulnerable (the reasoning being that the vulnerability was not publicly known and hopefully not yet actively exploited).

Not recommended means that a Git version was superseded by a patch release. We donâ€™t recommend using Git versions that are not at the latest patch level.

Recommended Git versions are the latest patch-level releases for which no vulnerabilities were reported before a specific date.
