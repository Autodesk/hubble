Tokenless Authentication Requests

If your GitHub Enterprise appliance is configured to use LDAP for authentication, then every request against GitHub requires an additional request against LDAP. At best, this will delay the GitHub request because GitHub needs to wait for the LDAP request first. At worst, the GitHub request will fail if the LDAP request fails (e.g. because the LDAP server is overloaded with requests). To avoid these problems users should use Personal Access Tokens (HTTPS) or SSH keys to authenticate. GitHub can check those without an additional request.
